🔧 Google Drive API 설정 가이드

⚠️ 현재 오류: redirect_uri_mismatch
Google Cloud Console에서 올바른 리다이렉트 URI를 설정해야 합니다.

📋 1단계: 현재 접속 URL 확인

현재 접속 중인 URL:

JavaScript가 로드되지 않았습니다.

위 URL을 Google Cloud Console에 추가해야 합니다.

🌐 2단계: Google Cloud Console 접속

Google Cloud Console 열기

Google Cloud Console > APIs & Services > Credentials로 이동합니다.

🔑 3단계: OAuth 2.0 클라이언트 ID 설정

  1. 기존 OAuth 2.0 클라이언트 ID를 클릭하거나 새로 생성
  2. "승인된 JavaScript 원본" 섹션에서 "URI 추가" 클릭
  3. 다음 URL 추가: http://localhost:8082
  4. ⚠️ 중요: "승인된 리다이렉트 URI" 섹션에는 아무것도 추가하지 마세요!
  5. JavaScript 인증 방식은 리다이렉트 URI가 필요하지 않습니다
  6. "저장" 버튼 클릭
⚠️ 중요: 현재 사용하는 JavaScript 인증 방식(Google Identity Services)은 리다이렉트 URI가 필요하지 않습니다! 오직 "승인된 JavaScript 원본"에만 현재 서버 주소를 추가하면 됩니다.
🚨 현재 오류: redirect_uri_mismatch
🔧 완전한 해결 방법:
  1. 1단계: Google Cloud Console → APIs & Services → Credentials
  2. 2단계: 기존 OAuth 2.0 클라이언트 ID 삭제
  3. 3단계: 새로운 OAuth 2.0 클라이언트 ID 생성
  4. 4단계: 애플리케이션 유형: "웹 애플리케이션" 선택
  5. 5단계: "승인된 JavaScript 원본"에만 http://localhost:8082 추가
  6. 6단계: "승인된 리다이렉트 URI" 섹션은 완전히 비워두기
  7. 7단계: 생성 후 새로운 클라이언트 ID를 앱에 적용
  8. 8단계: 브라우저 캐시/쿠키 완전 삭제 후 테스트

💡 핵심: 기존 OAuth 클라이언트를 수정하는 것보다 새로 생성하는 것이 확실합니다!

📝 4단계: API 키 및 클라이언트 ID 복사

  1. 클라이언트 ID를 복사 (형식: xxxxxxxxx.apps.googleusercontent.com)
  2. API 키를 복사
  3. 달력 앱에서 "☁️ 구글 드라이브 설정" 버튼 클릭
  4. 복사한 정보를 입력

🔄 5단계: 설정 적용

설정을 저장한 후 몇 분 기다린 다음:

  1. 브라우저 새로고침 (Ctrl + F5)
  2. "☁️ 구글 드라이브 설정" 버튼으로 인증 시도
✅ 성공하면: "🎉 Google Drive 인증 완료!" 메시지가 표시됩니다.
